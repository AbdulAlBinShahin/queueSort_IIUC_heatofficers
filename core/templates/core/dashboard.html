How the engine reasons

  1. Match the complaint to a transaction (amount, time, type, counterparty, status).
  2. Render an evidence verdict: consistent, inconsistent, or insufficient_data.
  3. Classify case type using priority-ordered English + Bangla keyword rules.
  4. Score severity from case type and amount.
  5. Route to one of six internal departments.
  6. Draft agent summary, next action, and customer reply with safety guardrails.
  7. Escalate to human review whenever the case is risky or unclear.
