Two endpoints. JSON in, JSON out.

Send the ticket payload. Get a structured analysis. No LLM. Synthetic data only.

Health check

GET /health

$ curl http://localhost:8000/health
{"status": "ok"}

Analyze a ticket

POST /analyze-ticket

Validates the payload, runs the rule-based engine, persists the analysis, and returns a single structured JSON object.

Request body

{
  "ticket_id": "TKT-0001",
  "complaint": "I sent 2500 BDT to 01712345678 yesterday but the receiver says they got nothing.",
  "language": "en",
  "channel": "in_app_chat",
  "user_type": "customer",
  "campaign_context": "",
  "transactions": [
    {
      "transaction_id": "TXN-001",
      "timestamp": "2025-01-15T14:32:00Z",
      "type": "transfer",
      "amount": 2500,
      "counterparty": "01712345678",
      "status": "completed"
    }
  ]
}

Response fields

  • matched_transaction_id - best-match transaction or null
  • evidence_verdict - consistent | inconsistent | insufficient_data
  • evidence_summary - one-sentence rationale
  • case_type - phishing | payment_failed | wrong_transfer | duplicate_charge | refund_request | agent_cash_in | merchant_settlement | other
  • severity - critical | high | medium | low
  • department - fraud | payments | dispute_resolution | merchant_ops | compliance | support
  • agent_summary - 1-2 sentence summary
  • recommended_next_action - operational next step
  • customer_reply - safe reply in the ticket language
  • human_review_required - boolean
  • confidence - 0.00-1.00
  • reason_codes - list of short identifiers

Example call

$ curl -X POST http://localhost:8000/analyze-ticket \
    -H "Content-Type: application/json" \
    -d @sample.json

Errors

  • 400 - validation error or prompt-injection attempt
  • 500 - analysis failed (generic message; no stack trace)
